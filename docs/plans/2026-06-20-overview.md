# Plan 2026-06-20 — pi-task-master: autonomous task orchestration over the pi SDK

## Goal

Build a model-agnostic **autonomous task orchestrator** on top of the pi agent
SDK — the same shape as `claude-task-master`, but provider-neutral and built
around pi. Give it a goal; it plans, executes, commits, pushes, opens PRs, fixes
CI failures, addresses review comments, and (configurably) merges. State
persists between runs so it resumes where it left off. Dynamic plan updates can
be injected mid-run via a mailbox.

The differentiator over `claude-task-master`: **per-phase model routing**. The
planner can run on a strong model (Claude Sonnet 4.6 / GPT-5.4), execution on a
cheap model (glm-5.2 / deepseek-v4-flash), and the CI fixer on a multimodal
model (GPT-5.4 / o3) when UI diffs need vision. claude-task-master is
Claude-only; this is the reason to build rather than adopt.

## Architecture

```
pitm start "goal"           # plan → execute → PR → (CI/review later) → done
pitm resume                 # reload state.json, continue from recorded phase
pitm status                 # show task/PR/phase state
pitm doctor                 # check pi auth, gh, git, config, models
pitm steer "change of plan" # write to mailbox → injected into running session
```

### State machine

```
planning → working → pr_open → ci_pending → ci_fixing → review → merging → verifying → done
                                                                                      ↘ needs_human
```

Phase 1 (this build) implements: `planning → working → pr_open → done`.
`ci_pending`/`ci_fixing`/`review`/`merging` exist in the type but route to
`needs_human` with an explicit "Phase 2/3/4 not built" message.

### Modules

1. **Orchestrator** — owns the state machine; one phase at a time; durably
   checkpoints `state.json` at every transition.
2. **Planner / Worker** — thin wrappers over `createAgentSession` with a
   per-phase model + scoped system prompt + per-phase tool set. (Fixer /
   Reviewer / Verifier land in later phases.)
3. **State** — single `.pitm/state.json`.
4. **Mailbox** — entries in `state.json`; orchestrator calls `session.steer()`.
5. **Doctor** — verify pi auth, `gh auth status`, git config, config file,
   model availability via `ModelRegistry.getAvailable()`.
6. **CLI** — small hand-rolled subcommand dispatcher (no extra dep).

### Model routing (config)

`.pitm/config.json` (repo-local):

```json
{
  "models": {
    "planner":  "anthropic/claude-sonnet-4-6",
    "worker":   "opencode-go/glm-5.2",
    "fixer":    "openai-codex/gpt-5.4",
    "reviewer": "anthropic/claude-sonnet-4-6",
    "verifier": "opencode-go/glm-5.2"
  },
  "verifyCommand": "bun run verify",
  "git": { "targetBranch": "main", "autoPush": true, "autoMerge": false },
  "budget": { "maxTokensPerRun": 2000000, "maxCiFixRetries": 3 }
}
```

Resolution: config → `ModelRegistry.find(provider, id)` → fallback to
`getAvailable()[0]`. `doctor` fails fast on a routed model that's unavailable.

## Phases (build order)

- **Phase 0 — Scaffold + doctor.** Package skeleton, CLI, config loader,
  `pitm doctor`. No agent calls.
- **Phase 1 — Minimal slice (the 80%).** `pitm start` → planner → worker →
  commit → push → `gh pr create` → `pitm status`. Durable state + `pitm resume`.
  Ship milestone.
- **Phase 2 — CI loop.** `gh pr checks --watch` → bounded fixer → `needs_human`.
- **Phase 3 — Review loop.** Poll reviews → reviewer session → push.
- **Phase 4 — Mailbox + merge + verify.**
- **Phase 5 — Multi-instance + concurrency.**

## Done when (Phase 1)

- `pitm start "<goal>"` on a feature branch: produces a plan, implements each
  task, runs `verifyCommand`, commits with a conventional message, pushes,
  opens a PR via `gh`.
- `pitm resume` after a `SIGINT` mid-worker picks up at the recorded phase.
- `pitm status` shows goal, phase, current task, PR url.
- Planner and worker run on the two different models configured in
  `.pitm/config.json` (verified from the run log).
- `pitm doctor` passes on this machine.

## Risks

- **SDK API stability** — pin pi version (`@earendil-works/pi-coding-agent@0.79.8`).
- **"Is the task actually done?"** — until Phase 4, human PR review is the gate.
- **Runaway worker** — per-run token `budget` hard cutoff; bounded attempts.
- **Vision in fixer** — glm-5.2 can't read images; route fixer to a multimodal model.
- **Auto-merge** — default `false`; orchestrator never merges without explicit config.
